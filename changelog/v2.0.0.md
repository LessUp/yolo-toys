# YOLO-Toys v2.0.0 Release Notes

**Release Date:** 2025-11-25

---

## 🎯 Multi-Model System

This release introduces comprehensive multi-model support, enabling dynamic switching between YOLO variants and HuggingFace Transformers models.

---

## ✨ New Features

### Multi-Model Support System

| Type | Models | Description |
|------|--------|-------------|
| YOLO Detection | YOLOv8 n/s/m/l/x | Object detection (80 COCO classes) |
| YOLO Segmentation | YOLOv8-seg series | Instance segmentation with pixel masks |
| YOLO Pose | YOLOv8-pose series | 17-keypoint human pose estimation |
| Transformer | DETR ResNet-50/101 | End-to-end object detection |
| Open Vocabulary | OWL-ViT | Zero-shot detection with text queries |
| Multimodal | BLIP Caption | Automatic image description |
| Multimodal | BLIP VQA | Visual question answering |

### New Backend Components

- `app/model_manager.py` — Unified model manager
  - Multiple model category support
  - Dynamic model switching
  - Model caching and auto device selection
  - Unified inference interface

### API Enhancements

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/models` | List all available models by category |
| `GET` | `/models/{model_id}` | Get model details |
| `POST` | `/infer` | Enhanced with `text_queries` and `question` params |
| `POST` | `/caption` | Image captioning endpoint |
| `POST` | `/vqa` | Visual question answering endpoint |

WebSocket now supports runtime configuration updates.

### Frontend UI Redesign

- Modern dark/light theme design
- Model category tabs for quick switching
- Top quick model selection dropdown
- Real-time performance overlay
- Image caption result display area
- Toast notification system
- Enhanced error handling

## 💡 Usage Examples

### Switch Models

1. Select model category tab in settings panel
2. Click model card to select specific model
3. Or use top quick selection dropdown

### Multimodal Features

1. Select "Multimodal" category
2. Image Captioning: Select BLIP Caption model, auto-generates description
3. Visual QA: Select BLIP VQA model, input question in settings

### Open Vocabulary Detection

1. Select OWL-ViT model
2. Input objects to detect in multimodal settings (comma-separated)
3. Example: `person, red car, dog`

---

## ⚠️ Notes

- HuggingFace models require download on first use, ensure network connection
- Multimodal models require more memory, GPU recommended
- Original YOLO functionality fully backward compatible
