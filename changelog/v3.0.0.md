# YOLO-Toys v3.0.0 Release Notes

**Release Date:** 2026-02-13

---

## 🏗️ Architecture Refactor

This release introduces a major architectural overhaul with Strategy Pattern implementation, significantly improving code maintainability, extensibility, and test coverage.

---

## ✨ New Features

### Strategy Pattern Architecture

New `app/handlers/` module implementing Strategy Pattern:

| File | Purpose |
|------|---------|
| `base.py` | BaseHandler abstract class with unified `load()`/`infer()` interface |
| `registry.py` | HandlerRegistry factory + MODEL_REGISTRY metadata |
| `yolo_handler.py` | YOLO detection/segmentation/pose handler |
| `hf_handler.py` | DETR / OWL-ViT / Grounding DINO handlers |
| `blip_handler.py` | BLIP Caption / VQA handlers |

### Pydantic Settings Configuration

New `app/config.py` using pydantic-settings:
- `.env` file auto-loading support
- Type validation for all configuration
- Derived properties (`origins_list`, `max_upload_bytes`)

### Enhanced Testing

Test cases increased from 6 to 16+:
- WebSocket connection/inference/config update tests
- Model info endpoint tests (404/200 scenarios)
- Boundary input tests (invalid content-type, empty file, missing parameters)
- HandlerRegistry unit tests
- AppSettings configuration tests

### Modern FastAPI Patterns

- **Lifespan context manager** replacing deprecated `@app.on_event("startup")`
- **Route extraction** to dedicated `routes.py`
- **Structured logging** replacing print statements

---

## 🔧 Changes

### Backend Refactoring
- `model_manager.py`: 814 lines → ~100 lines (pure delegation layer)
- `main.py`: 410 lines → 91 lines (initialization and middleware only)
- Global `print()` replaced with structured `logging`

### Frontend Optimization
- URL building extracted to `buildInferUrl()`/`buildWsUrl()` functions
- Eliminated URL construction code duplication
- Settings storage key version updated

### Infrastructure
- `docker-compose.yml`: Removed deprecated `version` field
- `.gitignore`: Enhanced with `.env`, IDE configs, model weights (`*.pt`)
- `.dockerignore`: Excluded tests/docs/changelog from Docker context
- `pyproject.toml`: Added `[project]` metadata, updated ruff config
